# Releasing

Releases are cut locally with `scripts/release.sh` (a wrapper around
[`cargo-release`](https://github.com/crate-ci/cargo-release)). Pushing the
resulting `vX.Y.Z` tag triggers the GitHub Actions **Release** workflow
(`.github/workflows/release.yml`), which uses
[`cargo-dist`](https://github.com/axodotdev/cargo-dist) to build prebuilt
binaries, publish a GitHub Release, and update the Homebrew tap
(`dodgog/homebrew-tap`).

## Prerequisites

All tooling is installed system-wide via the user's nix-darwin config at
`~/hub/git/config/flake.nix` (`pkgs.rustup`, `pkgs.cargo-release`,
`pkgs.cargo-dist`). No per-project dev shell.

If `cargo` errors with *"no default toolchain"*, run once:

```sh
rustup default stable
```

This downloads the stable Rust toolchain into `~/.rustup/toolchains/`. After
that `cargo`, `rustc`, `rustfmt`, `clippy` all work.

## Before releasing

Update the **`[Unreleased]`** section of `CHANGELOG.md` so the GitHub Release
notes reflect what shipped. The release script will not do this for you.

## Cut a release

From a clean `main`:

```sh
# Dry-run (default)
scripts/release.sh patch        # 0.3.3 → 0.3.4
scripts/release.sh minor        # 0.3.3 → 0.4.0
scripts/release.sh major        # 0.3.3 → 1.0.0
scripts/release.sh 0.5.0        # explicit version

# Execute for real
scripts/release.sh patch --execute
```

The script will:

1. Run preflight: `cargo fmt --check`, `cargo clippy -- -D warnings`,
   `cargo test`, `dist plan`. Skip with `--skip-checks` if needed.
2. Hand off to `cargo release`, which (per `release.toml`):
   - Bumps the version in `Cargo.toml` and `Cargo.lock`.
   - Bumps the `version = "..."` line in `flake.nix` via
     `pre-release-replacements`.
   - Commits `chore(release): X.Y.Z`.
   - Tags `vX.Y.Z` and pushes commit + tag to `main`.

Pushing the tag triggers the Release workflow on GitHub, which runs
`cargo-dist`, uploads platform binaries, creates the GitHub Release, and
publishes a Homebrew formula.

Watch the run:

```sh
gh run watch
gh release view vX.Y.Z
```

## Local builds (optional)

```sh
cargo build --release            # → target/release/sqs
nix build .                      # → ./result/bin/sqs
./result/bin/sqs --version
```

## Files

- `release.toml` — `cargo-release` config: branch, commit/tag format,
  `flake.nix` version replacement.
- `scripts/release.sh` — local entry point with preflight checks; dry-run
  by default.
- `dist-workspace.toml` — `cargo-dist` config: target platforms,
  installers, Homebrew tap.
- `.github/workflows/release.yml` — **autogenerated by `cargo-dist`**;
  do not hand-edit. Regenerate with `dist init` after changing
  `dist-workspace.toml`.
